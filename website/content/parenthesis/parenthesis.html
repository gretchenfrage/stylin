This is beautiful: 

y = (5 * x) + 9;

This, not so much: 

y = x*cos(10.0*pow(x,2.0))/(pow(x,2.0) + 1.0);

I've written worse lines myself. You can make sense of it if you squint, but it's hard to scan. Particularly, it's hard to tell the lexical depth at a glance.

>How can we make this readable?

I've adopted a style for managing these sort of bulky expressions: 

y = (
    (
        x
    ) * (
        cos(10.0 * pow(x, 2.0))
    ) / (
        pow(x, 2.0) + 1.0
    )
);

In bullet points:

  • Split into terms which make manageable one-liners
  • Put these terms in parenthesis
  • Every other line holds these one-liners
  • And the other lines hold the operands which combine them
  • Then the whole thing goes in parenthesis (plays nicer with other syntax)

Note that I used discretion here; I could have split this line up more or less. For example, I could make it look like this: 

y = (
    (
        x
    ) * (
        cos(
            (
                10.0
            ) * (
                pow(
                    x,
                    2.0
                )
            )
        )
    ) / (
        pow(x, 2.0) + 1.0
    )
);

But I think that goes too far, and makes the code harder to read. Use discretion. Remember: humans are the adaptable element in any system.
